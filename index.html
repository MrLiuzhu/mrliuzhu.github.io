aboArray

题目

移除数组 arr 中的所有值与 item 相等的元素，请直接在给定的 arr 数组上进行操作

测试样例：
  removeWithoutCopy([1, 2, 2, 3, 4, 2, 2], 2)
返回：
  [1, 3, 4]

javascript代码

function removeWithoutCopy(arr, item) {
    var len=arr.length;
    for(var i=len-1;i>=0;i--){
        if(arr[i]==item)
            {
                arr.splice(i,1);
            }
    }
    return arr;
}

javascript代码2

function removeWithoutCopy(arr, item) {
    var len=arr.length;
    for(var i=0;i<len;i++){
        if(arr[i]==item)
            {
                arr.splice(i,1);
            }
        i--;
    }
    return arr;
}

splice函数说明

  array.splice(start,delCount,item1,item2….);//该函数从start位置开始删除delCount数量个元素，然后从start的位置插入后面的其他元素
start:开始删除元素的位置：包括这个位置；
delCount:删除元素的个数；
item1,item2…:删除后从start位置开始添加的元素

例：var arr=[1,2,3,4];
arr.splice(1,2);//[1,4] 从位置为1的位置开始删除两个元素；
arr.splice(1,1,3);//[1,3,3,4] 从位置为1的地方删除一个元素，然后在这个位置添加元素3；

抛小球

题目

小东和三个朋友一起在楼上抛小球，他们站在楼房的不同层，假设小东站的楼层距离地面N米，球从他手里自由落下，每次落地后反跳回上次下落高度的一半，并以此类推知道全部落到地面不跳，求4个小球一共经过了多少米？(数字都为整数)

测试样例：
  100,90,80,70
返回：
  1020

c++代码

class Balls {
public:
    int calcDistance(int A, int B, int C, int D) {
        int sum=A+B+C+D;
        return 3*sum;
    }
};

分析

设每一个小球的初始高度为m,则每一个小球的第一次落地后，每次跳起至落地走过的路程为前一次的跳起高度，于是
Sn=m+m*(1+1/2+1/4+…+(1/2)^n)
Sn=m+m*((1-(1/2)^n)/(1-1/2))
当n趋近与无穷大时
Sn=3m

求两个正整数的最大公约数和最小公倍数

辗转相除法

辗转相除法， 又名欧几里德算法（Euclidean algorithm）,乃求两个正整数之最大公因子的算法。它是已知最古老的算法， 其可追溯至3000年前。

原理

javascript实现代码

function (m,n) {
    if (m<n) {
        temp=m; 
        m=n;
        n=temp;
    };
    do{
        temp=m%n;
        m=n;
        n=temp;
    }while(temp);
    var yue=m;              //最大公约数
    console.log(yue);
}

最小公倍数的值可以通过最大公约数求得，即为输入的两个整数的乘积在除以最大公约数得到的值。

OGC和OGC规范

OGC

OGC全称Open Geospatial Consortium，自称是一个非盈利的、国际化的、自愿协商的标准化组织，它的主要目的就是制定与空间信息、基于位置服务相关的标准。这些标准就是OGC的“产品”，而这些标准的用处就在于使不同厂商、不同产品之间可以通过统一的接口进行互操作。
在GIS领域，OGC已经是一个比较“官方”的标准化机构了，它不但包括了ESRI、Google、Oracle等业界强势企业作为其成员，同时还和W3C、ISO、IEEE等协会或组织结成合作伙伴关系。因此，OGC的标准虽然并不带有强制性，但是因为其背景和历史的原因，它所制定的标准天然地具有一定的权威性。

OGC标准

OGC的标准基本上就是OGC所有的成果，而所谓的标准就是一些接口或编码的技术文档。不同的厂商、各种GIS产品都可以对照这些文档来定义开放服务的接口、空间数据存储的编码、空间操作的方法等。
除了正式发布的标准（OpenGIS® Standards ），OGC的工作成果还包括一些其它类型的文档，比如讨论稿（Discussion Papers ）、抽象规范（Abstract Specification ）、最佳实践文档（Best Practices Documents ）、OGC参考模型（OGC Reference Model (ORM) ）、白皮书（White Papers ）等。在OGC的工作中，标准的制定可能由待讨论的“讨论稿”开始，在形成一定的统一意见后形成“抽象规范”，进一步具体化到“标准”，在这个过程中同时也可能形成“最佳实践文档”供印证，而“OGC参考模型”则是描述“抽象规范”、“标准”、“最佳实践文档”之间的关系。当然，其中我们最关心的还是OGC标准文档。
